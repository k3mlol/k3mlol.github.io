Preface

I have been work on a small party A company for almost 2 year, maybe I will leave this job soon.
I want to make a summary for my previous work. and talk about the architecture security design.
A good architecture can reduce the cyber security risk, make the security team fix the vulnerability more easily, help devops workmate have clear mind on architecture.

Ideas

  1. separate the (To client, To internal, To third party(maybe most is API)) network flow.

  2. using k8s. at old IT infrastructure, once the attacker can compromise a victim server, attacker can hit the intranet. k8s, there will no SSH service.
    the miscro service’s access can be more strict. especially once a server had been compromised, just repull the images again, the victim tool(webshell, mem shell)can be remove(in some case may be can’t, like mount the host volume).

security team have the limited time and energy, no company would like to hire many cyber security engineers. so us focus on the key point.

  1. pentest the internal service, especially find a attack path can hit the core service, show the impact to your workmate ,even in the intranet is not safe, the risk is can’t be ingored(if a company have a big technique debet, it’s quite hard to fix the vulnerability in short time).
    but remind the workmate that the put the service in the intranet is a temporary method.

monitor and protect

  1. monitor the k8s management service.
  2. We know that not all the services can move to k8s, so the k8s infrastructure server between other few services network connection will be easily ingored. if a server can stand run alone, let it go.

IT Governance

IT Governance is a world class problem, how to know a accurate amount of servers. how to know a accurate amount of miscro services. even a accurate amount of subdomain
